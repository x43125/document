# 常见问题处理

## 缓存穿透

频繁请求缓存中不存在的数据，导致缓存层失效，请求直接打到数据库，造成数据库压力

### 解决方案

1. 缓存 null 值，设置一个短时间的过期时间
2. 布隆过滤器

## 缓存击穿

热点 Key 过期，缓存重建过程中，大量请求直接打到数据库，造成数据库压力

### 解决方案

1. 热点key 不设置 Redis过期，由业务层主动过期，更新
2. 缓存重建时设置互斥锁，让同一时间只有一个请求去访问数据库，做重建操作

## 缓存雪崩

大量 key 同一时间过期或 Redis 宕机，导致大量请求直接达到数据库，造成数据库压力

### 解决方案

1. key 在设置过期时间时，设置随机时间，避免大量 key同一时间过期
2. 设置多级缓存，比如再加一层 ES
3. 熔断降级，太多的情况下直接熔断，避免把数据库打挂

## 热key

### 解决方案

1. 本地缓存：适用读多写少
2. 读写分离：加从库分担读压力，热key不频繁变化
3. 拆分：把一个key拆成多个，计数器场景
