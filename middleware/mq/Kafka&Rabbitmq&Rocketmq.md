# Kafka & Rabbitmq & Rocketmq

| 维度              | Kafka             | RabbitMQ         | RocketMQ          |
| :---------------- | :---------------- | :--------------- | :---------------- |
| **吞吐量**        | **百万级/秒**     | 万级/秒（5-10K） | 十万级/秒（50K+） |
| **延迟**          | <10ms             | 50-100ms         | 20-50ms           |
| **开发语言**      | Scala/Java        | Erlang           | Java              |
| **消息模型**      | Pull 拉取         | Pull + Push      | Pull + Push       |
| **顺序消息**      | 分区内有序        | 单消费者有序     | 队列内有序        |
| **事务消息**      | ✅ 支持            | ⚠️ 部分支持       | ✅ **原生支持**    |
| **延迟/定时消息** | ❌ 不支持          | ✅ 支持（AMQP）   | ✅ **支持**        |
| **死信队列**      | ❌ 不支持          | ✅ 支持           | ✅ 支持            |
| **消息重试**      | ❌ 不支持          | ✅ 支持（AMQP）   | ✅ 支持            |
| **消息过滤**      | ✅ 支持            | ✅ 支持（AMQP）   | ✅ 支持            |
| **优先级队列**    | ❌ 不支持          | ✅ 支持（1-9级）  | ❌ 不支持          |
| **多协议支持**    | ❌ 仅Kafka协议     | ✅ **7+种协议**   | ❌ 仅RocketMQ协议  |
| **管理界面**      | 需第三方工具      | ✅ 内置Web管理台  | ✅ 内置控制台      |
| **运维复杂度**    | 高（需ZooKeeper） | 中               | 中高              |
| **生态集成**      | **极丰富**        | 丰富             | 一般              |



> Java 项目，需要二次开发的无脑上RocketMQ



| 消息队列     | 一句话                                           |
| ------------ | ------------------------------------------------ |
| **Kafka**    | 吞吐之王，大数据和流处理首选，不适合业务消息     |
| **RabbitMQ** | 路由灵活，小而美，适合中小规模业务消息           |
| **RocketMQ** | 功能最全面，事务消息和延迟消息最强，电商金融首选 |

**选型原则：大数据看 Kafka，业务消息看 RabbitMQ，金融电商看 RocketMQ。没有最好的，只有最合适的**
