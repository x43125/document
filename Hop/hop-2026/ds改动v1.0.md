### 二、分布式工单系统数据架构重构（2023.08 – 至今）  
**角色**：核心架构与开发  

**技术栈**：ShardingSphere / MySQL / Canal / RocketMQ / Elasticsearch  

**项目背景**  
工单系统某版本上线后业务增长迅速，单月数据量突破 1500 万，单表达亿级，数据库 IO 成为瓶颈，核心接口响应耗时增加 300ms，慢 SQL 占比达 15%。

**核心工作与架构设计**  
1. **分库分表**  
   - 基于 ShardingSphere 实施物理分库 + 按月逻辑分表，根据业务维度选择分片键，避免跨分页查询，显著降低单表压力  

2. **MySQL + ES 异构架构**  
   - 构建 MySQL 负责核心写入、Elasticsearch 负责复杂查询的读写分离架构  
   - 通过 Canal 订阅 Binlog，经 RocketMQ 异步驱动实现准实时同步（延迟 < 2s）  
   - 设计补偿对账任务与幂等消费机制，保证异构数据最终一致性  

3. **查询优化**  
   - 在 ES 中针对工单文本、标签等维度建立倒排索引，利用 RoutingKey 优化分片查询  

**成果**  
- 核心多维检索接口平均响应时间从 3200ms 降至 80ms，性能提升 40 倍  
- 承载单月 1500 万增量数据，主库 CPU 负载峰值从 85% 降至 25%  
- 慢 SQL 基本清零，平稳支撑节假日大流量查询场景
