# Enmonster项目总结

## 消息中台

- 对标大厂的消息中台（不要觉得好像就是对接几个第三方 API，实际的开发中大家基本都是对接 API 或提供 API给别人用，不要妄自菲薄，而是要挖掘其中的闪光点）
- 核心：
  - 多渠道适配与稳定性
  - 高并发与性能
  - 消息可达性与重试机制
  - 模板与内容管理
  - 成本与配额管理
  - 安全与合规

## 公告系统

- 对标大厂的站内信系统：封装成站内信系统，主要讲设计理念，也是非常重要的一环，参考：B 站、微博、知乎等

## 智能客服系统

- AI 项目，客服不是重点，重点是“智能”
- RAG
  - 指标：召回率等
  - 更新
- Prompt 优化

## 审批流系统

- 看上去很简单，其实很复杂，可以好好梳理一下，如果有公司需要搭建的话，是很对口的，我们的审批流做的还是很完善的

## 登录系统

- 很核心的东西，可以说自己没有参与很多，但研究了很多

## 权限系统

- 很复杂
- 高并发
- 需要：严谨、高效

## 用户系统

- 高并发
