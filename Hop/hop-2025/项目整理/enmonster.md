*职责：负责多个微服务的日常维护与迭代开发及时响应用户线上及其他研发的问题处理等*

**项目一：**消息中台建设

**技术栈：**SpringCloud + Apollo + Mysql + Redis + RocketMQ

**背景**：现有反馈方式链路较长、维护麻烦且无法与内部系统直接关联，用户需要频繁切换系统，体验较差。

**技术方案：**

- 主导消息系统微服务架构设计与开发，基于Spring Cloud + RocketMQ + Redis 实现多通道消息模板配置，支撑多个业务模块快速接入，使得消息发送功能的交付周期从3人天缩短至0.5人天
- 通过Redis queue实现数据发送限流，保证数据不会因为API限速等无法发出，数据量可达每日100w，高峰可达150w

结果：消息系统稳步上线，日平均发送消息条数100w，异常错误率低于0.001%。

项目二：公共应用能力建设

简介：



















***\*技术栈：\****

\- SpringBoot, SpringCloud(Netflix)，Apollo配置中心做分布式基本架构。

\- RocketMQ做解耦，流量削峰，发布订阅等；Redis做缓存，分布式锁，队列，令牌桶限流，布隆过滤器等；ES做列表搜索，GEO查询，MySQL、Doris、OB等做基本数据库。

\- Activiti工作流，阿里云各种云能力对接包括：文件上传、阿里百炼AI平台等，腾讯云IM、客服系统。

\- Skywalking链路追踪、Granfana+Prometheus应用监控。



***\*职责：\**** 1.消息系统增加IM能力：通过策略模式及Redisson令牌桶算法实现对单接口限流（QPS<=200）；通过Redis、xxl-job实现消息消费、重试机制，消息发送成功率提升至99.99%，异常丢失率 < 0.01%；并不断迭代优化消息发送速度，目前将每10万条消息发送速度从52分缩小到10分内；

2.消息能力中台化：设计开发IM消息模板系统支持通过配置消息模板来控制如接收人、发送方式、消息样式等配置项，设计实现IM消息模版及自定义类型消息配置系统，业务接入消息系统效率提高70%。

***\*项目二：\****公众号系统 + 公告系统

***\*职责：\****  1.设计实现公众号系统：通过IM机器人+自定义消息模仿实现公众号消息类型，为app提供公众号能力。支持系统主动推送消息和用户被动拉取公告数据的两种方式；

2.开发独立通用的文章评论系统：结合AI实现自动审核及人工审核功能，设计交互事件提醒机制，并通过RocketMQ实现业务无侵入接入评论系统；
